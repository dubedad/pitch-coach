# Reference — the value-proposition recipe

The single most important thing a pre-seed deck must do is state a **clear value
proposition** — the narrative spine everything else hangs on. Rule 2 makes this the
litmus test: if the deck can't articulate a clean value-prop statement, no amount of
slide polish will save it. This file gives you the structure to test a deck's
positioning against, and the failure modes to name.

Use it as a **lens, not a script.** You test whether the founder's positioning holds
up; you never hand them a finished value-prop sentence (that would break Rule 0).

## The recipe (the structure a value prop must have)

> **For** [market segment / the stakeholders you deliver benefits to]
> **who** [the segment's KEY need],
> **our** [the thing or change you're "selling"]
> **provides** [the KEY benefit]
> **by** [HOW it delivers that benefit].
> **Unlike** [the status quo or leading alternative],
> **our** [thing] [the differentiator that is UNIQUE and can't be easily replicated].

The core principle: a value proposition **positions value relative to a real need
*and* to how that need is *currently* being met.** A benefit stated in a vacuum —
with no segment, no alternative, no differentiator — is not a value proposition. It's
a feature wearing a costume.

## The 8 ingredients (what has to be present)

1. **Segment** — a specific, identifiable, reachable "who." Not "everyone."
2. **Key need** — the one need that segment feels most sharply, stated as something
   observable, not abstract.
3. **The thing** — what is actually being sold (product, service, or change).
4. **Key benefit** — the primary outcome the segment gets, in their terms.
5. **How it's delivered** — the mechanism that makes the benefit real (not magic).
6. **The status quo / alternative** — what they use today. If there's "no
   alternative," the need probably isn't real.
7. **The differentiator** — why this is unique and hard to replicate. This is the
   moat in one clause.
8. **Positioning** — the whole thing framed *against* the alternative, so the value
   is relative and legible, not self-referential.

## How to use it when critiquing

- **Reconstruct the value prop from the deck.** Try to fill the recipe using only
  what's on the slides. Where you *can't* find an ingredient, that's the gap to hand
  back. ("I can see your segment and benefit, but I can't find what they use today or
  why you're hard to copy — so I can't tell if this is defensible.")
- **Name which ingredient is missing**, don't rewrite the sentence. Missing
  differentiator, missing "how," vague segment — each is a specific, quotable note.
- **Test for observability.** If the "key need" or problem isn't observable — you
  can't picture who hurts or what the gap is — say so and ask them to make it
  observable (this echoes the problem-slide standard in `deck-structure.md`).
- **The 3AM-infomercial test.** If a stranger couldn't repeat back *who it's for and
  why it beats the alternative* after one read, the positioning has failed — even if
  every slide looks polished.

## Failure modes to name

- **Benefit with no alternative** — "we save teams time" with nothing to compare
  against. Value isn't positioned; it's asserted.
- **Feature list masquerading as value** — describes what it *does*, never the *so
  what* for a specific segment.
- **"Everyone" as the segment** — no reachable customer, so no testable need.
- **No differentiator** — nothing that couldn't be copied by an incumbent in a
  quarter. Name the absence; don't invent the moat for them.
- **Undifferentiated "better/faster/cheaper"** — a comparative with no mechanism
  behind it ("by ___") is a claim, not a proposition.

## The line you never cross

You may point at *which* ingredient is missing and ask the question that surfaces it.
You may **not** write the value-prop statement for them, even as an "example." The
founder finds their own words; you find the hole.
